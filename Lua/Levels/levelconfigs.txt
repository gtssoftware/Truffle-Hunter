each level would be cfg. the level terrain itself must be a large png. then in the cfg, it would have this kind of data, to spawn an item or enemy, 
you would type in the pixel coordinates, 
then enter the directory of the enemies folder containing all its lua scripts. 
so when the level runs. it runs those lua scripts at the pixel coords. 
the hitbox would also be a png at the same res. in game the hitbox png is not rendered. the way it works
is that opaque pixels are treated as hitboxes while transparent ones arent. 
so it would look like this 

levelname('coolleveltest') 
levelterrain('IMG/TERRAIN/level1.png') 
levelsky('IMG/SKIES/sky6.png') 
levelhitboxes('IMG/HITBOX/LEVEL1/BOX.png')
spawnentity('Lua/enemies/blobenemy') 
levelend

all of that would be in something like coollevel.cfg
